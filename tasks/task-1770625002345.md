### 2. TYPOGRAPHY

A balanced and readable font pairing that communicates professionalism and a modern aesthetic.

*   **Heading Font:** **Poppins**
    *   **Weights:** Bold (700) for H1, SemiBold (600) for H2, Medium (500) for H3.
    *   **Usage:** Titles, headings, and prominent text elements. Its geometric and friendly nature makes it inviting for retail.

*   **Body Font:** **Open Sans**
    *   **Weights:** Regular (400) for general body text, SemiBold (600) for emphasis within paragraphs or short, impactful phrases.
    *   **Usage:** Paragraphs, descriptions, form labels, and all textual content requiring high readability. Its clean lines and excellent legibility ensure a smooth reading experience.

*   **Size Hierarchy:**
    *   **H1 (Main Headline):** 64px - 80px (Desktop), 48px - 60px (Tablet), 36px - 48px (Mobile)
    *   **H2 (Section Titles):** 40px - 56px (Desktop), 32px - 40px (Tablet), 28px - 36px (Mobile)
    *   **H3 (Sub-section Titles/Card Titles):** 24px - 32px (Desktop), 20px - 28px (Tablet), 18px - 24px (Mobile)
    *   **Body Text:** 16px - 18px (Desktop), 15px - 17px (Tablet), 14px - 16px (Mobile)
    *   **Small Text/Captions:** 14px (Desktop), 13px (Tablet), 12px (Mobile)
    *   **Line Height:** 1.5x - 1.6x for body text to ensure optimal readability.

### 9. ANIMATION REQUIREMENTS

Animations will be subtle, sophisticated, and serve to guide the user's eye and enhance engagement, not distract.

*   **Entrance Animations (Section-wise):**
    *   **Hero Section:** Content (Headline, Subheadline, CTA) fades in with a slight upward slide upon page load.
    *   **About Section:**
        *   Owner's image fades in.
        *   Text content slides in from the side (right for the text column).
    *   **Services/Products Section:** Cards animate in one by one or in staggered rows using a "stagger" effect, fading in and slightly scaling up.
    *   **Photo Gallery:** Images appear with a subtle fade-in and slight zoom-in effect as they enter the viewport.
    *   **Contact Section:** Form fields and map subtly fade in.

*   **Hover Effects:**
    *   **Buttons (All CTAs):**
        *   Slight scale-up effect (e.g., `transform: scale(1.05)`).
        *   Subtle background color change or intensification of the gradient.
        *   Brief shadow lift.
    *   **Cards (Services/Products):**
        *   Subtle lift with a more pronounced shadow on hover (`box-shadow: 0px 8px 20px rgba(0,0,0,0.15)`).
        *   Slight scale-up.
    *   **Social Media Icons:** Subtle color change or slight bounce animation.
    *   **Navigation Links:** Underline animation that draws out from the center or left.

*   **Scroll-Triggered Reveals:**
    *   Sections should animate into view as the user scrolls down the page. The animation should be smooth and not jarring.
    *   Elements within sections (e.g., images, text blocks) can also be configured to animate in as they become visible.

*   **Micro-interactions for Engagement:**
    *   **Form Field Focus:** Input border color changes smoothly to the accent color.
    *   **Button Click:** A subtle "press" animation on click.
    *   **Image Zoom on Hover (Gallery):** A slight, gentle zoom effect on images in the gallery on hover before click.
    *   **Navigation Highlight:** Active navigation link gets a subtle persistent glow or color change.

**Animation Libraries:** Consider using libraries like GSAP (GreenSock Animation Platform), AOS (Animate On Scroll), or CSS-native animations where appropriate for performance and maintainability. Animations should be optimized for smooth performance on all devices.

### 10. MOBILE RESPONSIVENESS

The website must be fully responsive and deliver an exceptional experience on all devices.

*   **Breakpoint Considerations:**
    *   **Mobile:** Max-width: 767px
    *   **Tablet:** Min-width: 768px and Max-width: 991px
    *   **Desktop:** Min-width: 992px
    *   **Large Desktop:** Min-width: 1200px (for richer layouts if needed, though standard desktop is sufficient)

*   **Layout Adjustments:**
    *   **Navigation:** Collapses into a hamburger menu on mobile/tablet.
    *   **Hero Section:** Text and CTA stack vertically and font sizes adjust for optimal readability.
    *   **Two-Column Layouts (About, Contact):** Stack vertically on mobile and tablet, with the left column appearing above the right.
    *   **Card Grids (Services/Products):** Adjust number of columns as described in Section 5.
    *   **Photo Gallery:** Adjust number of columns and image size.
    *   **Footer:** Columns stack vertically on mobile.

*   **Touch-Friendly Elements:**
    *   **Button Sizes:** Ensure all tappable areas (buttons, links, icons) are at least 44x44 pixels to prevent accidental mis-taps.
    *   **Form Fields:** Adequate spacing between fields, clear labels, and appropriate input types (e.g., `tel` for phone numbers).
    *   **Gallery Navigation:** Arrows in lightbox must be large enough to tap easily.

*   **Performance Optimization:**
    *   **Image Optimization:** Use responsive images (`<picture>` element or `srcset`) and compressed image formats (WebP).
    *   **Lazy Loading:** Implement lazy loading for images (especially in the gallery) to improve initial page load times.
    *   **Font Loading:** Optimize font loading strategies to prevent render-blocking.
    *   **Minification:** Minify CSS and JavaScript files.
